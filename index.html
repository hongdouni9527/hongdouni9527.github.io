正则表达式

发表于 2022-07-17 更新于 2023-02-04

正则匹配基础

  • 限定符：主要有*（0到多个），?（0或1个），+（1到多个），{n}（n个），{n,m}（n到m个），{n,}（n个以上）。限定符都紧跟在要限定的字符后面。比如yo+可以匹配”yooooooo…”，而yo{2,3}只匹配”yoo”或”yooo”。

  • 定位符：

    1. ^匹配开头，放在字符串开头。
    2. $匹配结尾，放在字符串结尾。
    3. \b边界，即字符和空格边界，单词一般左右两个边界，因此\b的位置决定了匹配那个边界。
    4. \B非边界，位置不重要，因为非边界并不会管你是哪个边界。

  • 范围：就是[]符号。此外，选择符号还可以用[0-9]表示全部数字，[0-9a-zA-Z]表示全部数字以及全部字母。值得一提的是[\u4e00-\u9fa5]可以匹配全部中文字符。此外，^符号在这里还有另外一个作用，就是否定，例如[^0-9]匹配所有非数字字符。

  • 选择：|表示选择，例如男主|女主，就可以将这两个都匹配到。[]在范围内选择是单字符，而选择给出了更大的自由。

  • 括号：我们已经见识到了[]（范围）和{}（限定）的作用，还有就是()符号了，括号提供了一个独立的空间，例如(男|女)主和前面是一样的，此处的选择操作就只发生在括号内部。对程序员还有更多用处，这里就不提及了。

  • 转义：有一点程序基础的应该会想到，前面定义了这么多特殊字符，应该要用到转义吧。一些非打印字符如\t\r\n\f等，依然保持原样，前面提到的特殊作用的字符，如果要匹配，需要在前面加上\转义，比如$匹配字符$。

  • 特殊字符：除了上面的，还有一些特殊字符以及特殊转义。

    1. .，匹配除了换行符\n之外任意单字符。
    2. \d，数字，等价于[0-9]；\D，非数字，等价于[^0-9]
    3. \w，任意数字，字母以及下划线。\W，\w之外的字符。
    4. \s，任意空白字符，等价于[ \f\n\r\t\v]，\S，任意非空字符。因此，通常可以使用[\s\S]来匹配任意字符。

鼠标滚动轮不够灵敏？ 经常上下抖动？

问题情况：鼠标使用一段时间后会出现滚轮不灵敏的情况，我的惠普鼠标就是如此，形成原因大概是由于灰尘或者其他污渍从滚轮口落入，进入了滚轮的电子器件中，导致检测不灵敏。

解决方法：解决鼠标滚动时页面上下跳动的问题_yzhuang的博客-CSDN博客_鼠标滚轮上下乱跳

实际操作：如博客教程所示，拆除后将铜片以及三个触点用棉签蘸取酒精进行清洗，同时将塑料片内的金属也进行擦拭，晾干后，重新组装即可恢复正常使用。（效果杠杠的！）

自建博客经验总结

发表于 2022-04-18 更新于 2022-04-20
1.搭建工具及搭建方法

​ 本次自建博客通过多方网络进行查询，最终决定采用半自动的方法，即github page + 博客工具hexo 进行搭建。后续应该会学习如何改善博客页面以及增添修改一些配置之类的。需要的软件应用为 git 和 Node.js 两个。同时在利用node.js 自带的npm函数进行hexo工具的安装。

​ 本次搭建主要参考了两份博客，一个是【基础篇】hexo博客搭建教程 - huanhao - 博客园 ，另一份是b站的搭建教程。

2.操作过程

​ 依照教程，简单的先安装Node.js 同时安装上git。在安装好Node.js 后进行系统路径配置，保证npm语句的全局调用，以及插件安装位置的设置。后面在安装hexo的过程中出现些问题，首先是通过CMD调用npm安装语句时报错提示需要管理员权限，结果查完后，有人说只要清除语句的缓存就可以，结果确实不在报错，但是hexo的安装位置错误，不在我要求的D盘，我找了半天，最终在C盘的user里面的Appdata里面的roaming 里面（痛苦面具）。经过一番尝试后我发现，不同通过清除缓存语句来解决管理员权限问题，因为清除缓存也就意味着之前设置的安装位置被删除。因此最终只能通过手动启动管理员的权限，然后在进行安装。

​ 安装完成后，我们需要在新建一个bolg文件夹，在这进行博客本地文件的初始化，所以需要在该文件下运行git bash ，结果照样需要管理员权限，因此又废了我一番功夫，将git以管理员方式打开。在初始化完毕后，我顺利的完成了第一个bolg文章的建立以及bolg结构的简单设置。最后只需要建立本地ssh秘钥，然后将它链接到github，保证可以通过hexo的一个上传工具将整个bolg文件提交就行。然而，事情并没有我想的这么简单，我也不知道我的电脑是抽了什么风，导致我输入生成ssh秘钥的指令始终无法执行，于是我又开始全网找文件，同时我甚至还怀疑到我在一开始下载blog模板时由于连接问题将一个网络链接端口进行修改，可能是这个导致指令无法执行。但是，渐渐的，没有找到相关问答后，我开始怀疑可能是我copy的问题，于是我又尝试了纯手敲的指令代码，并且准备在开管理员权限和不开管理员权限的程度下进行。最终，它突然就可行了，我也不知道原因，但我估计就是代码的空格或者啥的问题。

​ 最终，在生成的ssh秘钥与我的github账号下的一个仓库进行绑定，并且顺利上传bolg文件，同时进行bolg的架构设置。（这里有个点：很多以前的教程都没有更新的点，GitHub为了防止“黑命贵”运动的波及，将架构中的“master”改为了“main”，这也是你的博客在github page能不能顺利生成的重点之一。

​ 终于花费两天时间，我的博客成功搭建完成了，下一步就是学习如何编写博客，以及设置博客的高级运用。（当然你能看到这篇文章，也就说明我已经学会如何编写md文件（markdown文件）了！
